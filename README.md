# my-portfolio

## Projektbeskrivning

Projektet är en personlig portfolio som är till för att visa mina kunskaper inom CSS, HTML och JavaScript. Portfolion presenterar även information om mig och är tänkt att fungera som en grund för en vidareutvecklad portfolio längre fram.

## Syfte och mål

Syftet med projektet var att lära mig och tillämpa grundläggande webbutveckling med fokus på semantisk HTML, tillgänglighet och responsivitet. 
Målet har varit att skapa en enkel, tydlig och lättnavigerad portfolio som följer branschens principer.

## Metoder/Principer

SoC - Separation of Concerns

Målet har varit att dela upp uppgifterna enligt denna princip mellan HTML, CSS och JS. 
HTML ska enbart stå för vad innehållet ska vara. CSS ska stå för utseende och JS ska vara för funktionalitet och logik.

Utöver det har jag separerat CSS style sheets sektionsmässigt för att koden ska vara enklare att navigera och underhålla. Jag har även tänkt på en tydlig och separerad mappstruktur. 

## DRY - Don’t Repeat Yourself

DRY-principen har använts för att undvika upprepning. Exempelvis genom att återanvända klasser och stilar.

## Semantisk HTML

Projektet är uppbyggt med semantiska HTML-element såsom header, nav, main, section och footer för att förbättra struktur, tillgänglighet och läsbarhet i koden.

## Responsivitet

Jag har använt flex och grid för att göra webbplatsen mer responsiv mellan olika medier. Jag har utgått från desktop-first och därefter gjort media-queries för att anpassa innehållet efter mobila enheter.

## POUR
- Perceivable - Jag har hållt mig till enkelt, strukturerat innehåll med kontraster som är läsbara. Hover och focus har använts för att förtydliga interagerbara element.
- Operable - Det går att navigera webbsidan enkelt  med navbar. Navigation med tangentbord är möjligt, t.ex Tab eftersom elementen är strukturerade logiskt i HTML. JavaScript används endast som komplement, vilket gör att grundläggande funktionalitet fortfarande fungerar.
- Understandable - En tydlig layout med beskrivande rubriker och innehåll.
- Robust - Webbplatsen är byggd med semantisk HTML i åtanke, vilket gör att innehållet kan tolkas korrekt av webbläsare och hjälpmedel som skärmläsare. Den är någorlunda responsiv mellan desktop och mobila enheter.

## Förbättringar och Vidareutveckling

- Skapa enkla interaktions-animationer.
- Utökad responsiv design.
- Mer konsekvent stilval, t.ex val av färger, mellanrum mellan element och sektioner.
- Utökad och mer avancerad JavaScript-funktionalitet
- Fördjupat arbete kring tillgänglighet med t.ex ARIA-attribut.
